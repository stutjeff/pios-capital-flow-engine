# V6.6.2 修正摘要

- 核心資料品質改為「13 個模型因子」計數，不再把多個 backend 嘗試算進分母。
- 決策可信度不得高於資料完整度。
- 市場報酬改以有效交易觀測計算，避免週末／休市 forward-fill 產生假的 0.0%。
- 亞洲或區域資料不足時，跨市場傳導顯示「不可判定」，不再誤顯示 0/4。
- 歷史類比庫加入官方宏觀資料 fallback；舊事件若行情 API 不提供長歷史，仍可建立明確標記的 MACRO_FALLBACK 軌跡。
- GitHub Actions 移除 rebase 收尾，避免執行期殘留檔案再次阻擋 push。
- 新增 .gitignore 與 V6.6.2 專項測試。
